Welcome to yet another work-in-progress page. In this rewrite, I focus on

  • energy efficiency, making sure the animations can be cancelled and resumed at will, controlled by page blur.
  • custom element best practices, it should be able to control visual aspects of the visualization using nothing but cascading style sheets.
  • algorithmic complexity, the current implementation uses 0-level multilevel dynamics (ie... single level dynamis) because the two-level-dynamics code is broken. Once ready, it should display on the order of thousands of vertices instead of hundreds.
  • api consistency, trying to get the library adopted, I aim to write a consistent API. Similar functionality should be called in a similar manner.
